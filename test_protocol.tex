\section*{Overzicht}

Eén rijbeweging op- en neer, één rail, vier metingen + twee observaties.
Doel: elk DV1--DV3 criterium in de validatie-tabel van het verslag van
een echt cijfer voorzien in plaats van \texttt{[meet]}.

\textbf{Belangrijk: dit is gewoon de lift normaal omhoog en omlaag
rijden met de joystick/knoppen op de stoel.} Geen firmware-calibratie
mode (die \texttt{CalibrationRunService} flow in de code is voor
installatie, niet voor validatie). De firmware kent zijn rail al; wij
kijken alleen mee.

\subsection*{Wat we WEL testen}

\begin{itemize}[leftmargin=2em, label=$\bullet$]
  \item Of onze \texttt{IpcEncoderMapper} cumulatief de juiste positie
        oplevert door pulse-counts van straight + bend segmenten te
        integreren
  \item Of de Catmull-Rom interpolatie en quaternion-compositie van de
        chair op het scherm overeenkomen met de werkelijkheid
  \item Of het Kalman-filter encoder-ruis significant reduceert
  \item Of de Bluetooth round-trip binnen het 100\,ms budget blijft
  \item Of de hele pipeline live, end-to-end, soepel werkt
\end{itemize}

\subsection*{Wat we NIET testen}

\begin{itemize}[leftmargin=2em, label=$\bullet$]
  \item Encoder-hardware (al door DeVi geverifieerd)
  \item Firmware pulse-cost constanten (al geverifieerd)
  \item Lengte van bochten met meetlint (kan niet accuraat; lengte
        wordt door de firmware berekend uit pulse-count $\times$
        bend-radius $\times$ segment-hoek)
\end{itemize}

Dus: markers plak je \textbf{alleen op rechte stukken en bij de
stations}. Bochten meten we niet --- de software berekent ze, en
indirect testen we of die berekening klopt door te kijken of de chair
op de \emph{volgende} marker (op een rechte na de bocht) komt waar
verwacht.

\end{itemize}

\section{Logfile-capture}

De \texttt{TestLogger} autoload in de app schrijft alle validatie-events
automatisch naar een file op de tablet, in de app-private external
storage map (geen permissies nodig). Geen handmatig print-redirect of
\texttt{adb logcat} vereist tijdens de rijbeweging zelf. Je hoeft de
tablet niet met USB-kabel aan een laptop te koppelen tijdens het rijden
--- alleen \emph{na} de run om het bestand te pullen.

\subsection{Wat er automatisch gelogd wordt}

Zodra je de app opent:
\begin{itemize}[leftmargin=2em, label=$\bullet$]
  \item \texttt{[CSV]} per encoder-meting ($\sim$20 Hz tijdens beweging)
  \item \texttt{[BT\_RTT]} per request/response paar (round-trip in ms)
  \item \texttt{[BASELINE\_START]} / \texttt{[BASELINE\_END]} bij knop-druk
  \item \texttt{[MARKER]} per MARK-druk, inclusief encoder en positie
  \item \texttt{[VALIDATION\_SUMMARY]} bij Print summary
\end{itemize}

Bestand op de tablet:
\begin{lstlisting}
/storage/emulated/0/Android/data/com.devistairlifts/files/
    validation_logs/session_<datetime>.log
\end{lstlisting}

In de app onderaan de \emph{Validatie / Tests} sectie zie je de huidige
bestandsnaam en het aantal regels.

\subsection{Knop \emph{Nieuw logfile}}

Sluit het huidige bestand en opent een nieuw bestand. Counters van de
sessie worden ook gereset. Gebruik dit als je een nieuwe sessie wilt
starten (bv.\ na een testrun in demo-modus en vóór de echte run op de
lift).

\subsection{Na de rijbeweging: log pullen via adb}

Tablet via USB aansluiten op de laptop, dan:

\begin{lstlisting}
adb devices  # bevestig dat tablet zichtbaar is
mkdir -p ~/logs/tablet
adb pull /storage/emulated/0/Android/data/com.devistairlifts/files/\
validation_logs/ ~/logs/tablet/
\end{lstlisting}

Alle session\_*.log bestanden staan dan in \texttt{\textasciitilde/logs/tablet/}.

\subsection{Sanity-check vóór vertrek}

\begin{itemize}[leftmargin=2em, label=$\square$]

  \item Onderaan Validatie-sectie zie je: \emph{Log: session\_*.log (N regels)}
  \item Connect via \emph{Demo} entry → CSV samples teller in app loopt

  \item Klik \textbf{Start baseline} → status wordt oranje \emph{BASELINE ACTIEF}
  \item Klik \textbf{Nieuw logfile} → bestandsnaam in label verandert
\end{itemize}

\section{Eerlijkheids-regels}

\begin{itemize}[leftmargin=2em, label=$\square$]
  \item Bij failure (criterium niet gehaald): noteer de werkelijke waarde,
        markeer \emph{fail}, en bespreek de oorzaak in de Conclusion van
        het verslag. Niet ``een tweede run doen tot het past''.
  \item Eén run is genoeg voor proof-of-concept; N=1 is duidelijk in de
        Limitations vermeld.
  \item Bewaar de raw logfile in \texttt{logs/} (niet committen, te groot).
        Bewaar wel een geanonimiseerde samenvatting als \texttt{logs/run\_summary.txt}.
  \item Niets verzinnen. Als T1 niet gemeten is wegens ontbrekende
        instrumentatie, rapporteer dat: ``BT round-trip not measured
        in this run; instrumentation required for future validation.''
\end{itemize}
